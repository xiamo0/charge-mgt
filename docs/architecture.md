## 三、分层说明

| 层级 | 名称 | 职责 |
|------|------|------|
| **第一层** | 设备接入层 | 管理与充电桩的 WebSocket 连接，接收/发送 OCPP 消息 |
| **第二层** | 协议路由层 | 抽象多版本 OCPP 协议处理，厂商识别 |
| **第三层** | 消息处理管道 | 消息解析、协议转换、按厂商分类路由 |
| **第四层** | 云平台对接层 | 通过 Kafka 或 REST API 与云平台通信 |

## 五、技术选型

| 功能 | 方案 | 理由 |
|------|------|------|
| WebSocket | `tokio-tungstenite` | 异步高性能，与 tokio 完美结合 |
| Kafka | `lapin` | 纯 Rust AMQP 库，更易集成 |
| HTTP Client | `reqwest` | 异步 HTTP 客户端 |
| 配置 | `config` + `serde` | 统一配置管理 |
| 日志 | `tracing` + `tracing-subscriber` | 结构化日志 |

---

## 六、关键设计决策

| 决策项 | 选择 |
|--------|------|
| Kafka 客户端 | `lapin`（纯 Rust AMQP 库） |
| 消息发送模式 | 模式 A（充电桩消息 → Kafka → 云平台，异步解耦） |
| 厂商识别 | 从 BootNotification.chargePointVendor 获取 |
| Handler 异步化 | 改造为 `async fn handle` |

## 七、消息流程

```
充电桩 ─── WebSocket ───→ Connection
                                │
                                ↓
                    BootNotification 消息
                                │
                                ↓
                    解析 chargePointVendor
                                │
                                ↓
                    MessageRouter.route(vendor)
                                │
                                ↓
                    对应的 ProtocolHandler
                                │
                                ├──────────────────┐
                                ↓                  ↓
                         OCPP 响应           CloudMessage
                                │                  │
                                ↓                  ↓
                         发回充电桩          KafkaProducer.send()
                                                    │
                                                    ↓
                                              Kafka Queue
                                                    │
                                                    ↓
                                              云平台消费
```

## 八、实现阶段

### 第一阶段：基础设施
1. 添加依赖到 Cargo.toml
2. 实现 config.rs（配置结构）
3. 实现 error.rs（统一错误类型）
4. 实现 state.rs（连接状态管理）

### 第二阶段：设备接入层
5. 改造 ocpp-1-6 Handler trait 为异步
6. 实现 codec/json.rs（JSON 编解码）
7. 实现 device/websocket.rs（WebSocket 服务器）
8. 实现 device/connection.rs（连接处理）
9. 实现 device/handler.rs（消息处理入口）

### 第三阶段：协议路由层
10. 实现 protocol/trait.rs（ProtocolHandler trait）
11. 实现 protocol/ocpp16.rs（OCPP 1.6 协议处理）
12. 实现 protocol/router.rs（消息路由器）
13. 实现 protocol/vendor.rs（厂商识别逻辑）

### 第四阶段：云平台对接层
14. 实现 cloud/kafka.rs（Kafka 生产者）
15. 实现 cloud/client.rs（REST API 客户端）
16. 实现 cloud/message.rs（云平台消息结构）

### 第五阶段：集成与主流程
17. 实现 app.rs（整体协调）
18. 实现 main.rs（程序入口）
19. 编写单元测试
20. 运行 cargo clippy + cargo test --lib 验证
